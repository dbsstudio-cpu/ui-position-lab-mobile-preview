# 手機驗收清單

## 開啟前

- 使用無痕視窗或重新整理，避免看到舊快取。
- 記錄手機型號、OS 版本、瀏覽器與測試時間。
- 確認頁面顯示的 UI Position Lab 結果版本正確。

## 版面檢查

- Header 是否被瀏海、動態島或瀏覽器工具列遮住。
- 頂部與底部 Safe Area 參考線附近是否有元件過度貼邊。
- Bottom Navigation 是否壓到內容或離底部過遠。
- Home、Message Room、Compose 與 Input Frame 是否重疊或截斷。
- 卡片、按鈕與文字容器是否超出畫面。
- Dark 與 Soft Light 是否各自使用正確版面。
- 小尺寸與大尺寸比較時，間距是否出現明顯異常。
- 頁面是否出現非預期橫向捲動。

## 回報格式

```text
結果版本：
手機型號：
OS／瀏覽器：
主題：
頁面：
問題位置：
預期：
實際：
截圖檔名：
```

此清單只用於 UI 位置驗收，不記錄功能測試結果。
